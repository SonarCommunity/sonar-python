Why use named groups only to never use any of them later on in the code?

This rule raises issues every time named groups are:

  • referenced while not defined.

Noncompliant Code Example

import re

def foo():
    pattern = re.compile(r"(?P<a>.)")
    matches = pattern.match("abc")
    g = matches.group("b") # Noncompliant - group "b" is not defined
    g = matches.group(2) # Noncompliant - there is no group with this number

Compliant Solution

import re

def foo():
    pattern = re.compile(r"(?P<a>.)")
    matches = pattern.match("abc")
    g = matches.group("a")
    g = matches.group(1)
